# 简单系统监控系统 (Simple System Monitor)

## 项目概述

这是一个基于 TCP 和 HTTP 协议的分布式系统监控系统，包含客户端和服务端两部分。客户端运行在嵌入式设备上，负责收集系统信息并通过自定义协议发送到服务端；服务端接收数据并提供 Web 界面展示监控信息。

## 系统架构

```
+-------------------+      TCP      +------------------+      HTTP      +------------------+
|   Client (嵌入式)  | ==========> | TCP Server (主机) | ==========> | HTTP Server (Web) |
|                   |             |                  |              |                  |
| - 系统信息采集    |             | - 数据接收       |              | - 数据解析       |
| - TCP 发送        |             | - 文件存储       |              | - Web 展示       |
+-------------------+             +------------------+              +------------------+
```

## 功能特性

### 客户端 (Client)
- ✅ **系统信息采集**: 获取系统名称、版本、架构等基本信息
- ✅ **性能指标监控**: 实时监测 CPU、内存、磁盘、IO 使用率
- ✅ **运行时间统计**: 计算系统开机时间
- ✅ **TCP 数据传输**: 通过 TCP 协议发送监控数据到服务端
- ✅ **自定义协议**: 使用 FSTP (Custom Text Protocol) 传输数据

### 服务端 (Server)
- ✅ **TCP 服务**: 接收客户端发送的监控数据
- ✅ **数据存储**: 将接收到的数据和时间戳保存到共享文件
- ✅ **HTTP 服务**: 提供 Web 界面展示最新的监控数据
- ✅ **时间同步**: 向客户端返回当前时间戳

## 监控指标

- **系统信息**: 操作系统名称、版本、架构
- **CPU 使用率**: 实时 CPU 负载百分比
- **内存使用率**: 系统内存占用百分比
- **磁盘使用率**: 磁盘空间占用百分比
- **IO 使用率**: 磁盘 IO 负载百分比
- **系统运行时间**: 从启动至今的总时间

## 通信协议 (FSTP)

### 数据格式
```
UPDATE Name,<系统名称>,Version,<系统版本>,Arch,<系统架构>,Uptime,<运行时间>,CPU,<CPU使用率>,IO,<IO使用率>,Memory,<内存使用率>,Disk,<磁盘使用率>,time,<时间戳>\r\n
```

### 示例
```
UPDATE Name,Ubuntu,Version,18.04.6 LTS (Bionic Beaver),Arch,x86_64,Uptime,466595.22,CPU,1.18,IO,0.02,Memory,28.57,Disk,39.85,time,1765504916\r\n
```

## 工作流程

1. **客户端启动**: 初始化系统信息采集模块
2. **数据采集**: 定期获取系统性能指标
3. **协议封装**: 将数据按照 FSTP 协议格式打包
4. **TCP 传输**: 通过 TCP 连接到服务端并发送数据
5. **服务端接收**: TCP 服务端接收数据并添加时间戳
6. **数据存储**: 将接收的数据保存到 data.txt 文件
7. **HTTP 服务**: HTTP 服务端读取最新数据并提供 Web 访问
8. **Web 展示**: 客户端可通过浏览器访问监控数据

## 编译和运行

### 客户端编译
```bash
cd client/src
gcc -o ../bin/tcp_client tcp_client.c FSTP.c -lm
```

### 服务端编译
```bash
# TCP 服务端
cd sever/src
gcc -o ../bin/tcp_sever tcp_sever.c FSTP.c

# HTTP 服务端
gcc -o ../bin/http_sever http_sever.c
```

### 运行顺序
1. 先启动 TCP 服务端: `./sever/bin/tcp_sever`
2. 再启动 HTTP 服务端: `./sever/bin/http_sever`
3. 最后启动客户端: `./client/bin/tcp_client`

## 技术细节

- **网络协议**: TCP/IP, HTTP
- **编程语言**: C 语言
- **系统接口**: Linux 系统调用 (sys/utsname.h, sys/statvfs.h 等)
- **数据格式**: 自定义文本协议 FSTP
- **存储方式**: 文本文件 (data.txt)

## 适用场景

- 嵌入式设备远程监控

- 物联网设备状态上报
- 资源利用率统计分析

- 数据持久化存储便于历史分析
