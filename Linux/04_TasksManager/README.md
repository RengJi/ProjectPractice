# 多任务管理系统 (Task Manager System)

## 项目概述

这是一个基于 C++ 实现的命令行任务管理系统，采用面向对象设计和现代 C++ 特性，支持任务的增删改查操作，并具备数据持久化和日志记录功能。

## 核心特性

- ✅ **任务管理**: 支持任务的添加、删除、更新和查询
- ✅ **数据持久化**: 任务数据自动保存到本地文件
- ✅ **日志系统**: 完整的操作日志记录
- ✅ **排序功能**: 支持按 ID、优先级、截止日期排序
- ✅ **命令行交互**: 直观的命令行界面
- ✅ **CRTP 设计**: 使用 CRTP (Curiously Recurring Template Pattern) 实现静态多态
- ✅ **单例模式**: 日志系统采用单例模式设计

## 功能说明

### 支持的命令

| 命令 | 功能 | 语法示例 |
|------|------|----------|
| `add` | 添加任务 | `add 任务描述,优先级,截止日期` |
| `delete` | 删除任务 | `delete 任务ID` |
| `update` | 更新任务 | `update 任务ID,新描述,新优先级,新截止日期` |
| `list` | 列出任务 | `list [排序方式]` |
| `exit` | 退出系统 | `exit` |

### 排序选项

- `list` - 按 ID 排序（默认）
- `list 1` - 按优先级排序
- `list 2` - 按截止日期排序

## 技术架构

### 设计模式

1. **CRTP (奇异递归模板模式)**: 实现编译时多态，避免虚函数调用开销
2. **单例模式**: 确保全局唯一的日志记录器实例
3. **命令模式**: 将用户操作封装为命令对象

### 核心组件

#### 1. Task 结构体
- 存储任务的基本信息：ID、描述、优先级、截止日期
- 提供字符串转换功能

#### 2. TaskManager 类
- 管理任务的生命周期
- 提供增删改查接口
- 负责数据的加载和保存

#### 3. Command 系统
- 基于 CRTP 的命令基类
- 具体命令实现：AddCommand、DeleteCommand、UpdateCommand、ListCommand
- 使用智能指针管理命令对象

#### 4. Logger 系统
- 单例模式的日志记录器
- 线程安全的日志记录
- 操作记录持久化

## 编译和运行

### 编译步骤

```bash
# 进入项目目录
cd /root/03_project/02_TasksManager

# 创建构建目录
mkdir build && cd build

# 配置项目
cmake ..

# 编译
make

# 运行
./TaskManager
```

### 使用示例

```
> add 完成报告,3,2023-12-31
添加任务成功

> list
ID：1，描述：完成报告，优先级：3，截止日期：2023-12-31

> update 1 修改报告,5,2023-12-30
更新任务成功

> list 1
ID：1，描述：修改报告，优先级：5，截止日期：2023-12-30

> delete 1
删除任务成功
```

## 技术特点

- **C++17 标准**: 使用现代 C++ 特性
- **智能指针**: 自动内存管理
- **异常处理**: 安全的错误处理机制
- **线程安全**: 日志系统使用互斥锁保护
- **数据持久化**: 自动保存和加载任务数据
- **类型安全**: 强类型的命令参数解析

## 数据存储

- **task.txt**: 存储所有任务信息
- **log.txt**: 记录所有操作日志

## 扩展性

系统设计具有良好的扩展性：
- 可以轻松添加新的命令类型
- 可以扩展 Task 结构以支持更多属性
- 可以增强日志系统以支持更多功能
- 可以添加任务搜索和过滤功能
